## Panel 1: Agent Status Strip

### What it shows (per agent)

| Field | Source | Example |
|-------|--------|---------|
| Name | hardcoded | SAM |
| Role | hardcoded | Backend |
| Status dot | `AgentStatusIndicator` component | green/yellow/gray/red |
| Status label | computed from heartbeat | "Working", "Online", "Idle", "Offline" |
| Current task | `agents` table → `currentTask` | "AGT-328: Kanban Board" |
| Last heartbeat | `agents` table → `lastHeartbeat` | "2s ago" |

### Status logic (existing)

- **Online** (green): heartbeat < 15 min ago
- **Busy/Working** (yellow): heartbeat recent + has active task
- **Idle** (gray): heartbeat recent + no active task
- **Offline** (red): heartbeat > 15 min ago

### Existing code to reuse

- `AgentStatusIndicator` → status dot with colors + pulse
- `isAgentOnline()` → heartbeat threshold check
- `getStatusLabel()` → human-readable label
- `ceoMetrics.getAgentStatus` → backend query returns all agents with online/offline
- `agents.getAgentStatus` → per-agent detail query

### New work needed

**NONE for backend.** APIs exist.

**LEO (Frontend):**
- New `AgentStatusStrip` component — vertical list of agents
- Wire to `ceoMetrics.getAgentStatus` query
- Show current task from agent data
- Responsive: horizontal on mobile, vertical on desktop

## Panel 2: Live Terminal

### What it shows

Real-time execution log of agent tool calls. Like watching them code.

| Field | Source | Example |
|-------|--------|---------|
| Agent name | `executionLogs.agentName` | SAM |
| Tool call | `executionLogs.message` | `Read(convex/schema.ts)` |
| Level | `executionLogs.level` | info/warn/error |
| Timestamp | `executionLogs.timestamp` | "2s ago" |
| Metadata | `executionLogs.metadata` | exit code, files affected |

### Agent tabs

Top of terminal: `[MAX] [SAM] [LEO] [QUINN] [ALL]`

- Click agent name → filter to that agent's logs
- `ALL` → interleaved stream, color-coded by agent
- Default: `ALL`

### Existing code to reuse

- `ExecutionTerminal` component → already renders logs with tool colors, level indicators, auto-scroll
- `execution.getRecentLogs` → backend query with agent filter
- `executionLogs` table → full schema with level, message, metadata, agent

### New work needed

**SAM (Backend):** None. `execution.getRecentLogs` already exists with agent filtering.

**LEO (Frontend):**
- Wrap `ExecutionTerminal` with agent tab selector
- Add `ALL` mode with agent color-coding (use `AGENT_COLORS` from LiveDashboard)
- Auto-scroll to bottom on new logs
- Mobile: full-width, scrollable
